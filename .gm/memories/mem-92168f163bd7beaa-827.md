## netboot serve: two more live-Pi4 bugs fixed (dhcp-authoritative + world-readable tree)

(1) DHCP DISCOVER<->OFFER loop with NO REQUEST: Windows ICS (SharedAccess service, cant stop without admin) runs a 2nd DHCP server on the same eth0 link, so the Pi never accepted our offer. FIX: dnsmasq --dhcp-authoritative -> handshake completes DISCOVER->OFFER->REQUEST->ACK. (2) kernel panic unable-to-mount-root with NO initramfs: Alpine tarball ships boot/initramfs-rpi mode 600, cp -a preserves it, dnsmasq TFTP runs as nobody -> Permission denied -> kernel booted with no initramfs. FIX: chmod -R a+rX the netboot tree in build-netboot.sh + serve-netboot.sh. Both in serve-netboot.sh now. Pi halts at panic and needs POWER-CYCLE to retry each fix. dnsmasq log tell: sent .../initramfs-rpi = good, Permission denied = the 600 bug.
