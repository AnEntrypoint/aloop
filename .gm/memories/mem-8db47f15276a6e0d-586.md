ADR-013: aloop's audio worker thread heap-allocates AloopLoopDsp (the Faust home DSP struct) via std::make_unique instead of declaring it as a stack-local. sizeof(AloopLoopDsp) is 336,326,896 bytes (~320MiB, from 20 loopers x 60s delay-line buffers) -- no thread stack size can hold it as a local, causing a real SIGSEGV witnessed on a real Pi 4. Confirmed via gdb against a debug-build core dump (print sizeof(AloopLoopDsp) == 336326896). Two other hypotheses (8MiB explicit stack, memlock/ulimit -l) were tested and disproven first via live core dumps before the real cause was found.
