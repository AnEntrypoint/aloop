## netboot-dnsmasq.conf corrected to match live-witnessed serving

The reference host-side config src/net/config/netboot-dnsmasq.conf originally had tftp-unique-root (maps TFTP requests to a MAC subdir) — WRONG: the real Pi4 requests firmware under its BOARD SERIAL (7bec0617/start4.elf), so tftp-unique-root=mac never matches. Corrected: removed that directive, added dhcp-boot=bootcode.bin + standalone 192.168.137.x range + except-interface=lo + tftp-no-fail; documented that you serve flat and symlink <root>/<serial>/ -> root (serve-netboot.sh auto-does this). dnsmasq --test = syntax OK. image/serve-netboot.sh is the PREFERRED path (also runs the HTTP root server the initramfs needs); the conf is the dnsmasq-only reference. Commit d425932.
